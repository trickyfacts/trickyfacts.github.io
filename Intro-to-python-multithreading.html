Intoduction to python threading

Hello, guys. Now I am going to talk about python threading. It is a really interesting stuff. Hope you will like it.

What is threading?

Before we start working with threading, we have to learn what is thread. So now you have made a really interesting python programme. Like imagine you have made a code which prints "Hello" five times and prints "World" five times. and you have added them in two different functions. Like below

Fig1:The code where I didn't use threading

Here your computer has to do two tasks one is printing "Hello" for five times and printing "World" for five times. Each of these tasks are called thread. Our computer always handling a lot of threads. Like if you open your task manager you can see How many threads your computer is handling.

Fig2: The red circle indicates how many threads is being handled

Like in my computer, it is handling 2250 threads! Nowadays we have multicore cpus. It means we have more than one core in our cpu. You can also find it in your task manager. In my computer, I have 4 cores in my cpu.

By default in python, If you write a long code with a lots of tasks it will be considered as a single thread which is called "main thread". It means your code will run on a single of your cpu. As a result your code could lag, stop responding or also could run slow. But with multithreading we can actually divide our code and run it on different cores thus it will not lag and will run faster.
